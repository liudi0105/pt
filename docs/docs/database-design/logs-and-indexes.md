# 索引与日志

## 索引设计

重写时必须优先考虑查询路径，而不是只看字段是否完整。

### 1. 用户查询

常见查询：

- 按 id 查用户
- 按用户名查用户
- 按状态筛用户
- 按等级筛用户
- 按最近登录时间筛用户

### 2. 种子查询

常见查询：

- 按 id 查种子
- 按分类和状态查种子
- 按标签查种子
- 按发布时间和热度排序
- 按审核状态查种子

### 3. tracker 查询

常见查询：

- 按用户和种子查 peer
- 按种子聚合做种人数
- 按用户查完成记录
- 按状态清理过期 peer

### 4. 运营查询

常见查询：

- 按用户查签到、考核、H&R 和勋章
- 按状态筛选待处理记录
- 按时间窗口统计运营数据

## 日志设计

日志表不是附属品，是审计和排障的核心。

### 必须记录的内容

- 谁做了操作
- 操作了什么对象
- 操作前是什么状态
- 操作后是什么状态
- 发生时间
- 来源或上下文

### 常见日志类型

- 登录日志
- 改名日志
- 封禁日志
- 种子操作日志
- 考核日志
- H&R 日志
- 签到日志
- 奖励日志
- 插件操作日志

## 索引和日志的关系

- 索引用于提升查询速度
- 日志用于保留历史过程
- 两者都不应该替代主业务数据
- 只要状态会变化，就应该能在日志里追踪到

## 重写建议

- 高并发表优先设计组合索引
- 高筛选表优先设计状态索引
- 高历史表优先设计时间索引
- 高审计表优先设计对象索引
