# 维护策略

## 备份策略

数据库设计不仅要能写，还要能恢复。

### 备份对象

- 主库
- 附件文件
- 配置文件
- 搜索索引快照

### 备份要求

- 定期执行
- 可验证
- 可恢复
- 可回滚

## 数据增长策略

随着站点增长，数据会分化为三类：

### 1. 热数据

近期高频访问的数据，例如用户、种子、peer、消息。

### 2. 温数据

历史但仍常查的数据，例如签到、考核、H&R、日志。

### 3. 冷数据

不常访问但必须保留的数据，例如老日志、历史备份、归档附件。

## 生命周期管理

- 主业务表长期保留
- 日志表按规则归档
- 附件按业务生命周期管理
- 临时数据按任务清理

## 迁移原则

重写时数据库迁移要遵守：

- 先建核心表，再补充关联表
- 先稳定主对象，再扩展日志和统计
- 先保证写入正确，再优化查询
- 先保证恢复，再考虑性能调优

## 运维检查点

- 备份是否成功
- 恢复是否可用
- 慢查询是否可控
- 日志量是否异常
- 索引是否同步

## 重写落地要求

新的数据库设计至少要回答下面问题：

- 数据怎么分域
- 状态怎么表达
- 历史怎么保留
- 索引怎么加
- 备份怎么做
- 扩容怎么演进
