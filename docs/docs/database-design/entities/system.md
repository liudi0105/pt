## 核心实体说明

### settings

配置表应尽量支持：

- 按 key 读取
- 按模块分组
- 动态更新
- 缓存失效

#### 典型字段

- `id`
- `name`
- `value`
- `created_at`
- `updated_at`

#### 设计要点

- `autoload = yes` 的配置应优先进入缓存
- `value` 需要同时支持 JSON 和标量
- 配置项应按命名空间和模块组织

### agent_allows / agent_denies

客户端控制必须支持 allow/deny 组合策略。

建议保存：

- 客户端标识
- 规则说明
- 启用状态
- 命中范围

#### 设计要点

- allow 和 deny 不要混写在一张逻辑表里
- 命中规则最好可解释
- 客户端标识和备注信息要可追踪

### searchbox / searchbox_fields

搜索配置独立出来是为了支持：

- 自定义搜索入口
- 搜索字段扩展
- 高级筛选

## 使用场景

- 站点初始化和配置读取
- 风控与客户端限制
- 运营位和前台展示
- 搜索能力和字典管理
- 支持旧站兼容逻辑

## 风险点

- 系统表容易被“万能配置”污染，必须控制边界
- 字典表和业务表要分离
- 旧站兼容表要单独归档，避免影响主域理解
- 配置表读取频率高，必须有缓存策略
