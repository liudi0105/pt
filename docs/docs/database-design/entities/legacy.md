# Legacy 与支持实体

这一页归档不直接属于核心模块，但仍然属于数据库实体范围的表。

它们通常来自：

- 旧站兼容
- 审计和辅助
- 后台面板
- 推广和广告
- 站点历史

## 处理原则

- 这些表不一定都要重写，但必须被文档化
- 能合并到核心模块的，优先合并
- 纯旧站兼容的，单独标记
- 低频辅助表要说明是否保留

## 迁移建议

- 能并入新模块的实体，迁移时合并到主域
- 仅用于历史兼容的表，迁移时保持只读或归档
- 与 UI 相关的支持表，迁移时与前端资源一起处理
- 与风控相关的支持表，迁移时纳入安全模块

## 使用场景

- 旧数据迁移
- 兼容旧页面
- 后台支持功能
- 审计和安全补充

## 风险点

- legacy 表最容易被忽略，但迁移时最容易出问题
- 如果不单独文档化，后续会在清库、迁移和兼容阶段反复踩坑
- 旧表名称和新域名称可能不一致，文档里必须做好映射说明
