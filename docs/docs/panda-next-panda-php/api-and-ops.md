# 接口与运维说明

## 文档目的

这页不再做源码入口索引，而是说明站点有哪些接口族、它们分别服务什么业务、上线和运行时要关注什么。

## 接口族

### 1. 用户接口族

面向普通用户和前台页面。

覆盖内容：

- 登录、退出、个人资料
- 种子浏览和详情
- 评论、收藏、感谢、书签
- 消息、通知、投票、新闻
- 签到、奖励、完成记录

### 2. 管理接口族

面向管理员和运营人员。

覆盖内容：

- 用户管理
- 种子管理
- 设置和字典管理
- 考核和 H&R 管理
- 勋章、插件、风控和统计

### 3. Tracker 接口族

面向 BT 客户端。

覆盖内容：

- 下载上报
- 做种上报
- 完成上报
- 状态查询

### 4. 第三方接入口

面向自动化工具、外部脚本或联动系统。

覆盖内容：

- 读取登录态下的用户信息
- 与自动做种或辅助工具联动

## 接口设计原则

- 接口要按业务域划分，不按技术细节划分
- 普通用户接口与管理接口必须隔离
- Tracker 接口要高可用、低延迟、少依赖
- 外部接入口要限制能力和返回范围
- 所有写操作都要有审计痕迹

## 运维职责

### 1. 配置管理

包括：

- 时区
- 数据库
- Redis
- 搜索引擎
- 文件存储
- 站点开关

### 2. 任务运行

包括：

- 定时任务
- 异步队列
- 清理任务
- 统计任务
- 索引同步任务

### 3. 备份恢复

包括：

- 数据库备份
- 文件备份
- 恢复校验
- 历史保留策略

### 4. 日志监控

包括：

- 登录与权限异常
- tracker 失败请求
- 队列堆积
- 搜索同步失败
- 备份失败

## 上线检查项

- 用户登录正常
- 种子检索正常
- Tracker 上报正常
- 后台配置可保存
- 任务调度可执行
- 搜索和备份可用

## 运行检查项

- 日志是否增长异常
- 队列是否积压
- 索引是否延迟
- 备份是否成功
- 风控规则是否误杀

## 交付要求

后续重写接口时，每个接口族至少要写清楚：

- 输入是什么
- 返回什么
- 谁能调用
- 调用后改变什么状态
- 失败时怎么处理
