# 功能模块清单

这份文档用于重写前的功能梳理。目标不是解释代码入口，而是回答三件事：

1. 站点到底有哪些业务模块
2. 每个模块具体要做哪些功能
3. 重写时哪些能力必须首批具备

## 一级模块

| 模块 | 业务目标 | 主要对象 | 当前已识别能力 | 重写优先级 |
| --- | --- | --- | --- | --- |
| 账户与权限 | 建立用户身份、访问控制和账号治理 | 用户、邀请、登录态、权限 | 登录、登出、个人信息、用户管理、禁用/启用、重置密码、邀请信息、二步验证移除 | 高 |
| 种子与 Tracker | 管理资源生命周期和下载做种状态 | 种子、文件、Peer、Snatch、标签 | 种子列表/详情/发布/编辑、审核页、审核日志、审核动作、搜索框、Pieces Hash 查询、announce、scrape | 高 |
| 社区与运营 | 维持内容传播、互动和活跃治理 | 新闻、评论、消息、签到、投票、奖励 | 消息、评论、感谢、书签、新闻、投票、签到、通知 | 高 |
| 运营约束与激励 | 管理考核、H&R、勋章、求种和认领 | 考核、考核用户、H&R、勋章、奖励 | Exams、ExamUsers、H&R、Rewards、User Medals | 高 |
| 后台与治理 | 负责站点配置、风控、字典和运营面板 | 设置、客户端规则、标签、统计、插件 | Settings、Agent Allow/Deny、Dashboard、Tags、Medals、用户后台管理 | 高 |
| 数据与存储 | 提供业务数据、附件、索引和运行态存储基础 | MySQL、缓存、附件、日志、队列 | 数据库表体系、附件文件、运行日志、统计数据 | 高 |

## 主业务链路

### 1. 用户进入站点

- 注册或被邀请进入
- 登录并建立访问态
- 加载个人权限、等级、偏好和用户状态

### 2. 用户浏览和获取资源

- 浏览种子列表
- 使用搜索和筛选定位资源
- 查看详情、评论、感谢、收藏、书签
- 下载 `.torrent` 文件

### 3. Tracker 持续收集客户端状态

- 客户端 `announce`
- 站点返回 Peer 和种子状态结果
- 服务端记录下载、做种、完成、断开、速度等信息
- 形成 Peer、Snatch、H&R、统计和奖励输入

### 4. 社区与运营驱动活跃

- 发布新闻和公告
- 发站内消息和通知
- 用户签到、投票、互动
- 发放奖励、勋章和积分

### 5. 后台治理持续控制站点秩序

- 用户处理
- 种子审核
- 考核和 H&R 处理
- 客户端白名单/黑名单
- 全局配置和字典维护

## 重写时的模块边界

### 第一批必须具备

- 账户与权限
- 种子与 Tracker
- 社区基础互动
- 后台基础治理
- 数据与存储基础能力

### 第二批补齐

- 考核与 H&R
- 勋章和奖励
- 求种、认领、投诉申诉
- 更完整的消息和通知体系

### 第三批优化

- 搜索体验优化
- 插件与扩展能力
- 更细的统计面板
- 第三方系统接入

## 使用方式

- 产品拆分：按“前台功能 / 后台功能 / 状态流转 / 规则校验”拆需求
- 技术设计：按模块边界拆服务、接口和数据模型
- 排期实施：先交付首批主链路，再补运营治理能力
