# 功能模块清单

这是一份面向重写工作的功能地图。阅读方式是：先看模块是否存在，再看模块解决什么问题，最后按优先级拆分实现。

## 模块总表

| 模块 | 作用 | 关键子功能 | 依赖 | 优先级 |
| --- | --- | --- | --- | --- |
| 账户与权限 | 管理用户身份和访问控制 | 注册、登录、权限、封禁、资料、二步验证 | 认证、用户、日志 | 高 |
| 种子与 Tracker | 管理资源生命周期和做种状态 | 发种、审核、下载、做种、完成、搜索 | 种子、Peer、Snatch、索引 | 高 |
| 社区与运营 | 维持站点活跃和内容传播 | 新闻、投票、消息、评论、奖励、书签 | 社区内容、通知、积分 | 中 |
| 签到与考核 | 约束用户活跃度和站点贡献 | 签到、补签、考核、H&R、勋章 | 用户、日志、规则 | 高 |
| 认领与求种 | 管理资源需求与交付 | 认领、求种、结算、申诉 | 种子、用户、奖励 | 中 |
| 后台治理 | 统一配置和站点规则 | 设置、字典、风控、插件、统计 | 后台资源、配置、任务 | 高 |
| 搜索与索引 | 提供站内检索能力 | 搜索、索引同步、重建 | 搜索引擎、事件、任务 | 中 |
| 数据与存储 | 承载站点运行数据和附件 | 数据库、缓存、文件、备份 | MySQL、Redis、对象/文件存储 | 高 |

## 拆分原则

- 每个模块都要写清楚“为什么存在”
- 每个模块都要写清楚“用户看到什么”
- 每个模块都要写清楚“管理员怎么控制”
- 每个模块都要写清楚“状态怎么变化”
- 每个模块都要写清楚“它依赖什么、影响什么”

## 推荐实施顺序

### 第 1 阶段

- 账户与权限
- 种子与 Tracker
- 数据与存储

### 第 2 阶段

- 社区与运营
- 签到与考核
- 后台治理

### 第 3 阶段

- 认领与求种
- 搜索与索引
- 自动化任务和风控加固

## 总体说明

这套模块不是彼此独立的页面集合，而是围绕一条站点主链路组织：

1. 用户进入站点
2. 用户浏览和获取资源
3. tracker 持续记录下载和做种
4. 运营系统驱动活跃和治理
5. 后台与任务系统持续维护站点秩序
