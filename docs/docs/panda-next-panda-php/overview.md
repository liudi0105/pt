# 项目总览

`panda-php` 是一个完整的 PT 站点平台。重写时不要把它理解成“一个发种站”，而应理解成由以下几类能力组成的业务系统：

- 用户生命周期管理
- 种子发布与 tracker 跟踪
- 社区信息流与站内互动
- 签到、考核、H&R、勋章、奖励等运营机制
- 后台治理、配置、插件、搜索和运维自动化

## 重写目标

这套文档的目标是让开发可以直接据此开工，而不是只了解项目入口。

- 先知道有哪些功能模块
- 再知道每个模块解决什么问题
- 再知道模块之间如何联动
- 最后知道哪些模块必须优先重做

## 功能分层

### 1. 站点层

面向用户和管理员的功能集合，包含页面、交互和状态流转。

### 2. 业务层

围绕用户、种子、做种、考核、奖励、治理等核心对象建立规则。

### 3. 支撑层

搜索、缓存、队列、备份、外部存储、插件和日志。

## 功能优先级

重写时建议按下面顺序落地：

1. 账号、权限和登录态
2. 种子、Tracker 和做种统计
3. 社区信息流和站内消息
4. 签到、考核、H&R 和勋章奖励
5. 后台治理、配置和插件
6. 搜索和运维自动化

## 站点能力清单

- 账号注册、登录、验证、权限、封禁
- 种子发布、审核、标签、搜索、收藏、评论
- tracker 上报、做种统计、下载完成追踪
- 新闻、投票、消息、通知、奖励和感谢
- 签到、补签、考核、H&R、勋章、认领、求种
- 后台设置、字典、风控、插件、统计面板
- 全文检索、备份、队列任务和第三方接入

## 交付方式

后续文档采用“总索引 + 专题页”的方式组织。总索引只做导航和优先级说明，专题页负责写清楚每个模块的详细设计。
