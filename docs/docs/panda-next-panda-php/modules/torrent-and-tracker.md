# 种子与 Tracker

## 模块目标

这个模块负责管理站点最核心的资源生命周期。

它决定：

- 种子怎么被发布
- 谁能看到
- 谁下载了
- 谁在做种
- 谁完成了
- 哪些行为需要进入统计、奖励或处罚

## 功能清单

- 种子发布
- 种子编辑和维护
- 种子审核
- 种子拒绝与原因记录
- 种子标签、分类、来源、编码、音轨等元数据
- 种子附件与截图
- 下载、收藏、评论、感谢
- 做种状态追踪
- 完成记录和 snatch 记录
- 搜索与索引同步

## 生命周期设计

### 1. 发布前

发布前，资源处于待提交、待审或待处理状态。

要解决的问题：

- 资源格式是否完整
- 元数据是否满足站点规则
- 是否允许直接上架

### 2. 审核中

审核阶段要支持人工和规则判断。

审核动作包括：

- 通过
- 拒绝
- 要求修改
- 标记异常

### 3. 上架后

资源进入公开列表后，系统要支持正常访问和下载。

上架后要继续维护：

- 浏览量
- 下载量
- 做种人数
- 完成记录
- 搜索可见性

### 4. 下载与做种

tracker 是核心状态采集点。

系统需要持续知道：

- 谁在下载
- 谁在做种
- 谁完成了
- 谁掉线了
- 谁速度异常

### 5. 结算与归档

资源生命周期结束后，仍要保留行为记录。

结算对象包括：

- Snatch
- Peer
- H&R
- 积分和奖励
- 搜索索引

## 子功能详情

### 1. 种子发布

发布能力要保证资源信息完整。

需要支持：

- 选择分类和标签
- 填写基础元数据
- 上传附件或截图
- 配置是否需要审核
- 配置资源展示方式

### 2. 种子审核

审核是站点内容质量控制的核心。

需要支持：

- 审核列表
- 审核详情
- 审核通过
- 审核拒绝
- 拒绝原因记录
- 审核日志留痕

### 3. 用户下载与收藏

用户围绕种子产生的互动要被完整记录。

需要支持：

- 下载种子文件
- 收藏种子
- 查看详情
- 评论和感谢
- 书签和关注

### 4. tracker 上报

tracker 需要持续接收客户端状态。

需要支持：

- announce 上报
- scrape 查询
- 注册 peer
- 更新下载、做种、完成状态
- 处理断线和重连

### 5. 搜索与排序

种子检索是站点的主入口能力之一。

需要支持：

- 关键词检索
- 分类过滤
- 标签过滤
- 条件排序
- 热度和状态排序

## 关键规则

- 种子必须有明确状态
- 审核通过后才允许公开访问
- tracker 上报必须与用户和种子绑定
- 完成记录必须能回溯
- 异常客户端和异常速度需要进入治理逻辑

## 关键输出

- 公开的种子列表
- 审核结果
- tracker 记录
- 做种统计
- 完成记录
- 搜索索引

## 依赖模块

- 账户与权限
- 后台治理
- 数据与存储
- 搜索与索引
- 社区与运营

## 验收标准

- 种子可完整经历发布、审核、上架、下载、做种、结算流程
- tracker 能准确更新 peer 和完成状态
- 种子检索能返回正确结果
- 审核和拒绝动作可追踪
