# 社区与运营

## 模块目标

这个模块负责站点内容传播、用户活跃度维持和站点治理。

它解决的是一个问题：如何让站点不只是“能下载”，而是“能持续运转”。

## 功能清单

- 新闻和公告
- 投票和意见收集
- 站内消息和通知
- 评论、感谢、打赏、收藏、书签
- 签到和补签
- 勋章和奖励
- 考核机制
- H&R 处理
- 认领和求种

## 子功能详情

### 1. 内容信息流

站点需要一个稳定的内容发布能力，用来传达运营信息。

包括：

- 新闻发布
- 公告推送
- 投票发起
- 结果统计

信息流的目标不是社交网络化，而是支撑站点运营动作。

### 2. 站内互动

用户对内容和种子会产生一系列互动。

包括：

- 评论
- 感谢
- 收藏
- 书签
- 打赏或奖励

这些互动一方面提升活跃度，另一方面也是站点统计和推荐的输入。

### 3. 消息与通知

站内消息是用户和系统之间的重要沟通通道。

需要支持：

- 私信
- 系统通知
- 登录提醒
- 运营通知
- 未读状态管理

### 4. 签到系统

签到是站点最基础的活跃机制之一。

需要支持：

- 每日签到
- 连续签到
- 补签卡
- 签到日志
- 签到奖励

### 5. 勋章和奖励

奖励体系用于长期激励和身份识别。

需要支持：

- 勋章定义
- 勋章获取条件
- 用户勋章展示
- 奖励记录
- 积分变动记录

### 6. 考核机制

考核是 PT 站常见的站点规则约束。

需要支持：

- 考核规则配置
- 考核任务分配
- 完成进度跟踪
- 通过与淘汰
- 人工干预和复核

### 7. H&R 管理

H&R 负责追踪用户下载后不做种或做种不足的行为。

需要支持：

- H&R 记录生成
- 状态更新
- 豁免处理
- 批量清理
- 风险提醒

### 8. 认领与求种

认领和求种将“需求”和“交付”联系起来。

需要支持：

- 发布求种
- 创建认领
- 认领状态跟踪
- 完成结算
- 过期或失效处理

## 关键规则

- 签到和奖励必须可追溯
- 考核和 H&R 必须有明确状态
- 运营行为要能形成闭环，不允许“只发不管”
- 消息和通知必须有未读状态
- 人工处理动作要留痕

## 关键输出

- 站内通知
- 签到结果
- 奖励和积分变动
- 考核结果
- H&R 状态
- 认领和求种状态

## 依赖模块

- 账户与权限
- 种子与 Tracker
- 数据与存储
- 后台治理

## 验收标准

- 用户可以完成签到、查看奖励、接收通知
- 考核和 H&R 能正确反映状态变化
- 评论、感谢、收藏等行为能形成统计输入
- 运营动作可在后台追踪和处理
