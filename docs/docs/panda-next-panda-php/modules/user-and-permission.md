# 账户与权限

## 模块目标

这个模块负责定义“谁能进入站点、能看到什么、能做什么、失去什么资格”。

它不是单纯的登录注册，而是站点所有业务的入口控制层。

## 功能清单

- 注册、登录、退出
- OAuth 或第三方授权登录
- 密码管理和二步验证
- 用户等级、权限、状态控制
- 封禁、启用、重置密码
- 用户资料与个人偏好
- 登录日志和账号变更记录
- 邀请关系和邀请额度

## 子功能详情

### 1. 注册与初始化

新用户进入站点后，系统需要完成基础身份创建和默认配置初始化。

需要支持的能力：

- 创建初始账号
- 设置默认等级和初始状态
- 初始化语言、时区、通知偏好
- 记录来源和注册时间

### 2. 登录与会话

登录是站点访问控制的第一关。

需要支持的能力：

- 用户名密码登录
- 站点内会话保持
- API token 或授权态
- 退出登录
- 异常登录失败提示

### 3. 权限控制

权限不是单一角色判断，而是“等级 + 角色 + 状态 + 规则”的组合。

需要支持的能力：

- 普通用户、版主、管理员等不同等级
- 菜单、接口和页面访问控制
- 用户状态驱动的访问限制
- 后台管理权限控制

### 4. 账号状态管理

用户状态直接影响其能否参与站点行为。

常见状态包括：

- 正常
- 禁用
- 封禁
- 待验证
- 待处理

状态变化场景包括：

- 违规处罚
- 管理员人工干预
- 验证失败
- 安全风险处理

### 5. 资料与偏好

用户资料不仅是展示信息，也会影响站点体验。

需要支持的能力：

- 头像、昵称或展示字段
- 简介和个性信息
- 语言和时区偏好
- 封面图展示偏好
- 通知与展示设置

### 6. 邀请体系

邀请体系是 PT 站常见的用户增长和风控工具。

需要支持的能力：

- 邀请额度控制
- 邀请关系追踪
- 邀请记录和状态
- 邀请异常风控

## 关键规则

- 账号状态必须作为所有功能的前置校验
- 管理员操作必须留痕
- 敏感操作必须可回溯
- 登录态失效后不得继续访问受限资源
- 用户的等级变化必须同步影响权限能力

## 关键输出

- 站点可用的登录态
- 用户访问权限结果
- 用户个人状态视图
- 管理动作审计记录

## 依赖模块

- 数据与存储
- 后台治理
- 社区与运营
- 种子与 Tracker

## 验收标准

- 新用户能完整完成注册和登录
- 不同等级用户访问权限表现一致
- 被禁用账号无法继续使用受限功能
- 管理动作能够被日志追踪
