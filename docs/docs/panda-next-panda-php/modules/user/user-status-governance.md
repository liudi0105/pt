# 用户状态治理

## 功能目标

管理用户的可用性、处罚和恢复过程，确保用户状态变化能够立即影响站点行为。

## 参与角色

- 管理员
- 被处理用户
- 系统自动规则

## 入口

- 管理后台用户处理页
- 用户状态校验中间件
- 安全和运营触发动作

## 前台功能

- 用户查看自身状态结果
- 被限制时收到明确反馈

## 后台功能

- 禁用用户
- 启用用户
- 修改状态字段
- 写入管理备注
- 触发密码重置

## 状态定义

- 正常
- 待验证
- 禁用
- 封禁
- 警告中
- 待人工处理

## 状态流转

1. 用户初始进入默认状态
2. 因违规、安全或运营原因进入限制状态
3. 管理员启用、恢复或继续处罚
4. 状态变化立即影响登录和受限功能

## 规则与校验

- 任何限制状态都必须有来源动作
- 启用/禁用应记录操作人和时间
- 用户状态应作为所有核心功能前置校验
- 恢复状态后权限按等级重新生效

## 异常与边界

- 已禁用用户重复禁用
- 恢复后仍存在其它限制条件
- 批量处理部分失败

## 数据结果

- 用户当前状态
- 处罚与恢复记录
- 管理备注
- 操作审计

## 验收标准

- 用户状态可直接控制登录和核心业务访问
- 每次状态变更都有完整审计信息
- 恢复动作后用户行为权限恢复一致
