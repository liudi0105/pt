# 邀请体系

## 功能目标

承载 PT 站用户增长与准入控制能力，记录邀请关系、邀请额度和邀请结果。

## 参与角色

- 邀请人
- 被邀请人
- 管理员

## 入口

- 用户中心邀请信息页
- 后台邀请信息读取
- 注册流程中的邀请校验

## 前台功能

- 查看邀请额度
- 查看邀请记录
- 查看邀请结果

## 后台功能

- 读取邀请信息
- 处理异常邀请
- 管理邀请有效性

## 状态定义

- 未使用
- 已发送
- 已注册
- 已过期
- 已失效

## 状态流转

1. 邀请人获得额度
2. 发出邀请并生成唯一标识
3. 被邀请人注册或超时过期
4. 管理员可人工处理异常状态

## 规则与校验

- 邀请必须绑定邀请人
- 过期邀请不能继续注册
- 邀请结果要记录邮箱、用户名和注册用户
- 邀请异常需要可追溯

## 异常与边界

- 邀请码不存在
- 邀请已过期
- 预注册信息与最终注册信息不一致

## 数据结果

- 邀请关系
- 邀请额度消耗结果
- 注册来源事实

## 验收标准

- 邀请发出、使用、过期形成完整闭环
- 后台能准确查看邀请关系和结果
- 注册流程可正确识别无效邀请
