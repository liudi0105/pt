# 权限与等级

## 功能目标

定义用户在站点中“能看到什么、能进入什么、能执行什么动作”的规则基础。

## 参与角色

- 普通用户
- 特殊角色用户
- 管理员

## 入口

- 登录后权限加载
- 页面访问校验
- 接口访问校验
- 后台管理动作校验

## 前台功能

- 根据权限决定页面可见性
- 根据权限决定可执行动作

## 后台功能

- 查询等级列表
- 配置等级使用边界
- 按管理权限控制后台菜单和动作

## 状态定义

- 基础等级
- 特殊身份
- 有权限
- 无权限
- 因状态被剥夺权限

## 状态流转

1. 用户登录
2. 系统计算等级、状态、特殊身份
3. 输出页面和接口权限结果
4. 状态或等级变化后重新生效

## 规则与校验

- 权限不能只靠前端隐藏实现
- 用户状态优先于功能权限
- 管理员敏感动作必须校验操作者等级
- 特殊身份不能绕过禁用状态

## 异常与边界

- 等级存在但状态禁止访问
- 前端显示与后端权限不一致
- 管理员越权操作

## 数据结果

- 用户等级
- 权限判断结果
- 后台管理能力集合

## 验收标准

- 同一用户在页面和接口上的权限判断一致
- 禁用、封禁等状态可覆盖普通权限
- 后台越权操作被拒绝并留痕
