# 登录与认证

## 功能目标

建立用户进入站点和维持登录态的完整闭环，确保前台访问、API 调用、Passkey 登录和 OAuth 授权使用同一套身份事实。

## 参与角色

- 未登录用户
- 已登录用户
- 管理员
- 第三方接入系统

## 入口

- 登录页
- API 登录接口
- Passkey 登录链接
- OAuth 授权确认页

## 前台功能

- 用户名密码登录
- 登出
- 登录后读取当前用户信息
- OAuth 授权确认
- Passkey 快速登录

## 后台功能

- 管理员重置密码
- 管理员移除二步验证
- 查看登录相关日志

## 状态定义

- 未登录
- 已登录
- 登录失败
- 禁用用户拒绝登录
- 已建立授权态

## 状态流转

1. 用户提交凭证
2. 系统校验账号、密码、状态
3. 成功则建立会话或 Token
4. 返回当前用户上下文
5. 失败则返回明确失败结果，不建立登录态

## 规则与校验

- 未启用或被禁用账号不能登录
- 登录成功后必须返回用户等级、状态和关键权限结果
- 登出后原有受限接口不得继续访问
- Passkey 登录必须绑定唯一用户
- OAuth 授权必须要求已有登录态

## 异常与边界

- 密码错误
- 用户不存在
- 账号被禁用
- Passkey 无效或过期
- 第三方授权缺少回调上下文

## 数据结果

- 登录态
- 登录日志
- 最近登录时间
- 当前用户上下文

## 验收标准

- 登录、登出、当前用户读取形成闭环
- 禁用账号无法建立登录态
- Passkey 和 OAuth 不绕过用户状态校验
- 管理员重置密码后旧密码失效
