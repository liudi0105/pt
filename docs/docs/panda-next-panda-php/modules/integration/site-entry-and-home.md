# 站点入口与首页跳转

## 1. 概述

管理站点根路径（`/`）到前台首页、登录入口（`/login`）到登录页的跳转逻辑，确保用户首次访问时进入正确页面，登录完成后能回到原始目标页。

## 2. 功能说明

### 2.1 入口路由映射
| 请求路径 | 实际文件 | 说明 |
|----------|----------|------|
| `GET /` | `index.php` | 站点首页 |
| `GET /login` | `login.php` | 登录页面 |

系统通过简单的 PHP 入口文件路由，不涉及复杂路由规则。

### 2.2 访问流程
1. **未登录用户访问首页**：直接进入 `index.php`，看到前台首页内容。
2. **未登录用户访问受限页面**：系统将用户重定向至 `login.php`，并将原始目标页记录在 `returnto` 参数中。
3. **用户登录**：在 `login.php` 提交凭证验证通过后：
   - 如有 `returnto` 参数，跳回原目标页
   - 无 `returnto`，进入首页
4. **已登录用户**：直接访问任何页面，不触发登录重定向。

### 2.3 跳转约束
- 根路径和登录入口只做入口承接，不做业务判定。
- 跳转保留原始目标 URL 的可恢复性，不丢失或篡改。
- 避免首页到登录页的无意义循环（已登录用户直接跳过）。

### 2.4 登录重定向实现
登录页面验证用户身份后，通过 session 或 GET 参数中的 `returnto` 决定跳转地址。若无目标页或目标页不可访问，则默认跳转首页。

## 3. 操作入口

- `GET /` — 站点根路径
- `GET /login` — 登录入口
- `index.php` — 前台首页文件
- `login.php` — 登录页面文件

## 4. 使用说明

1. 站点根路径 `/` 配置在 Web 服务器（Nginx/Apache）中指向 `index.php`。
2. 登录入口 `/login` 同样指向 `login.php`，通过 Web 服务器路由规则实现。
3. 开发新页面时无需修改入口逻辑，直接创建对应 PHP 文件并在需要时添加路由映射。
4. `returnto` 参数来自用户访问受限页面时的登录拦截，值为编码后的目标 URL。
5. 登录后的跳转逻辑位于 `login.php` 的验证通过分支中。

## 5. 配置参考（可选）

- Web 服务器配置（Nginx location 规则 / Apache RewriteRule）定义 URL 到 PHP 文件的映射
- Session 配置影响登录态保持时长
