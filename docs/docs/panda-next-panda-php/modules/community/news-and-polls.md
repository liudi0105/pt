# 新闻与投票

## 1. 概述

提供站点公告发布和用户意见收集能力。新闻用于发布站内公告和运营活动信息；投票用于收集用户意见和社区决策。

## 2. 功能说明

### 2.1 新闻管理

管理员发布和编辑站内新闻。

- 数据模型 `News`：表 `news`，字段 `userid`（发布者）、`added`（发布时间）、`title`、`body`、`notify`（是否发送通知）。
- 新闻发布后按 `added` 时间倒序展示在前台新闻列表。
- `GET /api/news-latest`：获取最新新闻，同时更新该用户的 `last_home` 时间为当前时间，用于下次查询"未读新闻"数量。
- 当 `notify=1` 时，新闻发布后会在通知聚合接口中增加计数。
- 完整的 CRUD 支持：`/api/news` 提供列表、详情、创建、更新、删除操作。

### 2.2 投票管理

管理员创建和管理投票，用户参与投票并查看结果。

- 数据模型 `Poll`：表 `polls`，字段 `added`（创建时间）、`question`（投票问题）、`option0`~`option19`（最多 20 个选项）。
- 关联模型 `PollAnswer`：表 `pollanswers`，字段 `pollid`、`userid`、`selection`（选择的选项编号）。
- 前台展示每个选项的投票人数和占比。

#### 投票 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/polls` | GET/POST | 投票列表 / 创建投票 |
| `/api/polls/{id}` | GET/PUT/DELETE | 投票详情 / 更新 / 删除 |
| `/api/polls-latest` | GET | 最新投票（含用户已投选项和统计） |
| `/api/polls-vote` | POST | 提交投票（参数：`poll_id` + `selection` 0-255） |

#### 投票约束

- 每个用户对每个投票只能投一次。
- `selection` 值范围为 0-255，但实际有效选项为 0-19。
- 已结束的投票不再接收新投票。
- 投票结果包含每个选项的投票人数。

## 3. 操作入口

- 新闻列表页：`/api/news`
- 新闻详情页：`/api/news/{id}`
- 最新新闻接口（首页/侧栏）：`/api/news-latest`
- 投票列表页：`/api/polls`
- 投票详情页：`/api/polls/{id}`
- 投票操作：`/api/polls-vote`

## 4. 使用说明

1. 新闻发布后自动出现在前台列表，无需审核。
2. `GET /api/polls-latest` 会返回当前用户对该投票的选择记录（如已投）及各选项得票统计。
3. 用户提交投票后不可修改或撤销。
4. 投票选项最多 20 个（option0 ~ option19），超出部分无效。
5. 新闻的 `notify` 字段控制是否计入通知聚合，不影响前台展示。
