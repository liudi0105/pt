# 签到

## 1. 概述

每日签到功能通过连续签到奖励和补签机制维持用户活跃度，为用户提供稳定的魔力值（Bonus）收入来源。适用于所有已登录的普通用户。

## 2. 功能说明

### 2.1 每日签到

用户每天可签到一次，签到后系统发放魔力值奖励并记录签到日志。

- **首次签到**：发放初始奖励 `INITIAL_BONUS=10` 魔力值。
- **连续签到**：从签到日志中向前追溯连续天数 `days`，奖励计算公式：`points = min(INITIAL_BONUS + (days - 1) * STEP_BONUS, MAX_BONUS)`，其中 `STEP_BONUS=5`、`MAX_BONUS=1000`。
- **里程碑奖励**：连续签到达到特定天数时额外加成——第 10 天 `+200`、第 20 天 `+500`、第 30 天 `+1000`。
- **断签处理**：若历史签到不连续，则 `days` 重置为 1，`total_days` 继续累加。
- **操作结果**：增加用户 `seedbonus` 余额，写入 `Attendance` 记录（`uid`, `added`, `points`, `days`, `total_days`），创建 `AttendanceLog`（`uid`, `points`, `date`），并返回排名数据。

### 2.2 补签

允许用户消耗补签卡回溯补签最多 30 天内的缺失记录。

- 补签卡数量从用户魔力值余额中扣减，单价由配置 `bonus.attendance_card` 控制（默认 1000）。
- 补签时需指定目标日期，系统校验该日期在 30 天内且当日确实未签到。
- 补签成功逻辑：从目标日期开始计算连续天数、按签到奖励公式发放积分、扣除 1 张补签卡、创建标记 `is_retroactive=1` 的 `AttendanceLog`。

### 2.3 补签卡购买

- 调用 `BonusRepository::consumeToBuyAttendanceCard()` 执行扣费。
- 扣费成功后用户的补签卡库存 +1。

### 2.4 签到数据查询

- 用户可查看自己的签到日志列表和连续签到天数。
- 系统提供排名数据（如连续签到排行榜）。

## 3. 操作入口

- 签到页面：用户在每日首次进入时展示签到入口
- 签到记录页面：查看历史签到日志和连续签到统计
- API：`GET /api/attend` — 执行签到并返回结果

## 4. 使用说明

1. 用户每天仅可正常签到一次，当天已签到再次访问不会重复发放奖励。
2. 补签需确保有足够的补签卡库存，且目标日期在 30 天回溯期内。
3. 连续签到奖励有上限 `MAX_BONUS=1000`，达到后不再增长。
4. 所有签到记录和奖励发放均写入数据库，支持审计追溯。

## 5. 配置参考

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `bonus.attendance_card` | 1000 | 补签卡单价（魔力值） |
| `INITIAL_BONUS` | 10 | 首次/断签后基础奖励 |
| `STEP_BONUS` | 5 | 每多连续一天追加奖励 |
| `MAX_BONUS` | 1000 | 连续签到奖励上限 |
| `MAX_RETROACTIVE_DAYS` | 30 | 最大可补签天数 |
