# 签到

## 功能目标

通过每日签到和补签维持用户活跃，并形成奖励输入。

## 参与角色

- 普通用户
- 管理员
- 系统任务

## 入口

- 签到入口
- 签到记录页

## 前台功能

- 每日签到
- 使用补签卡
- 查看连续签到结果
- 查看签到奖励

## 后台功能

- 查看签到日志
- 处理异常签到记录

## 状态定义

- 今日未签到
- 今日已签到
- 已补签
- 连签中断

## 状态流转

1. 用户当天首次签到
2. 系统写入签到记录并发奖励
3. 如缺失某日且有补签卡，可执行补签
4. 连签统计随记录变化更新

## 规则与校验

- 每日只能正常签到一次
- 补签必须消耗补签卡
- 奖励必须与签到事实绑定

## 异常与边界

- 重复签到
- 无补签卡却尝试补签
- 跨时区导致日期歧义

## 数据结果

- 签到记录
- 连签结果
- 奖励变动

## 验收标准

- 正常签到和补签都能形成可追溯记录
- 连签统计与签到事实一致
- 奖励发放不会重复
