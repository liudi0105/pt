# 消息与通知

## 功能目标

建立用户与用户、系统与用户之间的核心沟通通道。

## 参与角色

- 普通用户
- 管理员
- 系统任务

## 入口

- 站内信列表页
- 未读消息区
- 通知中心

## 前台功能

- 查看消息列表
- 查看未读消息
- 阅读消息
- 删除消息
- 接收系统通知

## 后台功能

- 发送管理消息
- 触发系统通知
- 查看消息发送结果

## 状态定义

- 未读
- 已读
- 已删除
- 通知待送达
- 通知已送达

## 状态流转

1. 用户或系统生成消息/通知
2. 接收方看到未读状态
3. 阅读后变更为已读
4. 用户可删除或归档（按策略）

## 规则与校验

- 未读状态必须准确
- 删除消息不应破坏审计需要的系统消息事实
- 管理消息与普通私信边界要清晰

## 异常与边界

- 接收用户不存在
- 消息被删除后重复读取
- 通知重复触发

## 数据结果

- 消息记录
- 未读统计
- 系统通知记录

## 验收标准

- 用户能稳定接收、阅读和管理消息
- 未读数准确反映消息状态
- 系统通知与私信边界清楚
