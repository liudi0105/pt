# 考核与 H&R

## 功能目标

对用户行为施加持续性约束，管理考核任务、达标过程和下载后不做种的治理。

## 参与角色

- 普通用户
- 管理员
- 系统规则任务

## 入口

- 用户考核状态页
- H&R 结果页
- 后台考核管理页
- 后台 H&R 管理页

## 前台功能

- 查看考核状态
- 查看 H&R 状态
- 接收风险提醒

## 后台功能

- 创建和管理考核
- 匹配考核用户
- 豁免、恢复、批量处理
- 单条或批量处理 H&R

## 状态定义

- 考核待生效
- 考核进行中
- 考核通过
- 考核失败
- H&R 待处理
- H&R 已达标
- H&R 已赦免

## 状态流转

1. 管理员创建考核规则
2. 用户匹配进入考核
3. 系统持续更新进度
4. 达标、失败或人工豁免
5. H&R 由 Snatch/做种事实生成并持续更新

## 规则与校验

- 考核命中逻辑必须可解释
- H&R 必须基于真实下载/做种事实
- 豁免和恢复动作必须留痕

## 异常与边界

- 用户重复进入同一考核
- H&R 重复生成
- 批量处理部分失败

## 数据结果

- 考核定义
- 考核用户记录
- H&R 记录
- 治理审计日志

## 验收标准

- 考核和 H&R 都具备明确状态机
- 后台能单条和批量处理
- 前台看到的状态与后台处理结果一致
