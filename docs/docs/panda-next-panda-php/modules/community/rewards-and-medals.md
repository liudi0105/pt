# 奖励与勋章

## 1. 概述

实现用户间的魔力值打赏（Reward）和勋章系统（Medal）。打赏用于用户间正向激励，勋章用于身份展示和成就积累。

## 2. 功能说明

### 2.1 打赏（Reward / Magic）

用户消耗自己的魔力值打赏给种子发布者，表达对资源的认可。

- 数据模型 `Reward`：表 `magic`，字段 `torrentid`、`userid`、`value`（打赏金额）。
- **打赏流程**（事务内执行，使用乐观锁控制并发）：
  1. 前置校验：
     - 打赏者魔力值余额充足。
     - 未对该种子重复打赏。
     - 种子存在且状态正常。
     - 种子发布者账号状态正常。
     - 不能打赏自己发布的种子。
  2. 创建 `Reward` 记录，扣减退打赏者魔力值，增加种子发布者魔力值。
- API：`GET /api/rewards?torrent_id=X`（查询种子收到的打赏）、`POST /api/rewards`（执行打赏，参数 `torrent_id` + `value`）。

### 2.2 勋章（Medal）

勋章系统包含定义、颁发、佩戴和管理四个环节。

#### 2.2.1 勋章定义

- 勋章分为两种获取类型：
  - **Exchange（兑换型）**：用户消耗魔力值主动兑换。
  - **Grant（授予型）**：管理员直接授予指定用户。
- 每个勋章定义包含：名称、图标、获取方式、库存数量、限购数量、售价（魔力值）、销售时间窗口（开始/结束时间）、有效期等。
- 系统配置：`system.maximum_number_of_medals_can_be_worn` 控制用户最多可同时佩戴的勋章数量。

#### 2.2.2 兑换流程

1. 用户浏览勋章商店，选择可兑换的勋章。
2. 系统校验：用户尚未拥有、勋章在销售时间窗口内、库存充足、用户魔力值足够。
3. 扣除魔力值，创建 `UserMedal` 记录，关联用户和勋章。

#### 2.2.3 佩戴与展示

- 用户可在个人资料页选择已拥有的勋章进行佩戴。
- 佩戴数量受系统配置限制（`maximum_number_of_medals_can_be_worn`）。
- 勋章有有效期（`expire_at`），过期后不再展示。
- 同一时间只能佩戴 `UserMedal` 中 `is_wearing=1` 的勋章。

#### 2.2.4 管理后台

- 管理员可创建/编辑/删除勋章定义。
- 管理员可手动为用户授予或移除勋章（Grant 类型）。
- 支持查询所有用户的勋章持有和佩戴状态。

## 3. 操作入口

- 打赏功能：种子详情页打赏按钮
- 勋章商店：勋章列表页查看可兑换勋章
- 用户中心勋章页：佩戴/卸下勋章
- 后台管理：勋章管理、用户勋章管理
- API：
  - `GET /api/rewards?torrent_id=X`、`POST /api/rewards`
  - `resource /api/medals` — 勋章 CRUD
  - `resource /api/user-medals` — 用户勋章关系管理

## 4. 使用说明

1. 打赏不可撤回，操作前请确认金额无误。
2. 同一用户不可对同一种子重复打赏。
3. 兑换型勋章消耗魔力值后不可退款。
4. 佩戴勋章数量有上限，超限时需要先卸下已有勋章。
5. 过期勋章自动失效，不再出现在用户资料展示中。

## 5. 配置参考

| 配置项 | 说明 |
|--------|------|
| `system.maximum_number_of_medals_can_be_worn` | 用户最多可同时佩戴的勋章数量 |
| 各勋章 `price` | 兑换型勋章的魔力值售价，在勋章定义中独立配置 |
