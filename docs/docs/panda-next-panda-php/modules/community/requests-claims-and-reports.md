# 求种、认领、举报

## 1. 概述

提供用户发布资源需求（求种）、认领需求并交付资源、以及举报违规内容的能力。形成从需求提出到交付完成的运营闭环。

## 2. 功能说明

### 2.1 求种（Request）

用户发布资源需求，描述需要的资源信息。

- 数据模型 `Request`：表 `requests`，字段 `userid`、`request`（标题）、`descr`（详细描述）、`comments`（评论数）、`hits`（浏览数）、`added`（发布时间）。
- 求种发布后可在前台列表中展示，其他用户可以浏览和评论。
- 基础 CRUD 操作，无复杂业务逻辑。

### 2.2 认领（Claim）

用户认领求种需求，承诺制作并发布对应资源。

#### 2.2.1 认领流程

- 数据模型 `Claim`：表 `claims`，字段 `uid`、`torrent_id`、`snatched_id`、`seed_time_begin`、`uploaded_begin`、`last_settle_at`。
- 认领前提条件：
  - 目标种子已存在且发布时间超过 `TORRENT_TTL=30` 天。
  - 用户确实下载过该种子（存在 snatch 记录）。
  - 用户当前认领数未超过 `USER_UP_LIMIT=10`。
  - 该种子被认领次数未超过 `TORRENT_UP_LIMIT=1000`。
- 认领时记录当前做种时间（`seed_time_begin`）和上传量（`uploaded_begin`）作为基准。

#### 2.2.2 认领结算

定时任务对认领记录进行周期性结算：

- **达标判定**：对比当前做种时间和上传量与认领时基准值的增量。
  - 做种时间标准：`STANDARD_SEED_TIME_HOURS=300` 小时。
  - 上传量标准：上传量增量 ≥ 基准上传量 × `STANDARD_UPLOADED_TIMES=2`。
  - 每月判定：`seed_time_this_month` 和 `uploaded_this_month` 为月度累计值，`is_reached_this_month` 标记是否达标。
- **达标奖励**：发放魔力值奖励并发送结算站内信。
- **未达标处罚**：
  - 认领超过 1 个月仍未达标：扣除 `REMOVE_DEDUCT=600` 魔力值，移除认领记录。
  - 首个自然月未达标：保留认领记录，继续观察。
- **主动放弃**：用户可主动放弃认领，扣除 `GIVE_UP_DEDUCT=400` 魔力值。

#### 2.2.3 管理后台

- Filament `ClaimResource` 提供后台管理界面，支持管理员查看、移除、调整认领记录。

### 2.3 举报（Report）

举报违规内容（种子、评论等）。

- 数据模型 `Complain`：当前版本为裸模型，大部分业务逻辑在旧版 PHP 代码中实现。
- 用户提交举报后，管理员在后台查看和处理。

## 3. 操作入口

- 求种列表和详情页：浏览和发布求种需求
- 认领管理页：执行认领、查看认领状态
- 举报入口：各资源页面的举报按钮
- 后台管理：Filament ClaimResource 管理认领记录
- API：求种相关 CRUD 接口、认领相关接口

## 4. 使用说明

1. 认领后系统以月度为单位进行结算，上月未达标不影响下月继续努力。
2. 主动放弃认领需扣除魔力值，请谨慎操作。
3. 认领种子必须是已发布超过 30 天的种子，新种子不可认领。
4. 用户最多同时认领 10 个种子，单个种子最多被认领 1000 次。

## 5. 配置参考

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `TORRENT_TTL` | 30 天 | 种子发布多久后可被认领 |
| `USER_UP_LIMIT` | 10 | 单个用户最多同时认领数 |
| `TORRENT_UP_LIMIT` | 1000 | 单个种子最多被认领次数 |
| `STANDARD_SEED_TIME_HOURS` | 300 | 月度达标做种时间标准 |
| `STANDARD_UPLOADED_TIMES` | 2 | 月度达标上传量倍数标准 |
| `REMOVE_DEDUCT` | 600 | 超期未达标扣除魔力值 |
| `GIVE_UP_DEDUCT` | 400 | 主动放弃扣除魔力值 |
