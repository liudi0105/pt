# 求种、认领、举报与申诉

## 功能目标

承载资源需求、交付承诺和异常治理的运营闭环。

## 参与角色

- 需求发起用户
- 认领用户
- 举报用户
- 被举报/被申诉用户
- 管理员

## 入口

- 求种列表和详情页
- 认领处理页
- 举报与申诉入口
- 后台处理页

## 前台功能

- 发布求种
- 创建认领
- 查看状态
- 发起举报
- 发起申诉

## 后台功能

- 管理求种和认领状态
- 处理举报
- 处理申诉
- 记录处理结果

## 状态定义

- 求种待响应
- 已认领
- 已完成
- 已过期
- 举报待处理
- 申诉待处理
- 已处理

## 状态流转

1. 用户发布需求或举报/申诉
2. 其它用户或管理员接手处理
3. 状态进入处理中
4. 最终完成、关闭或失效

## 规则与校验

- 状态变更必须有处理主体
- 举报和申诉必须保留处理结果
- 认领完成需明确交付事实

## 异常与边界

- 重复认领
- 对不存在对象举报
- 申诉对象不存在

## 数据结果

- 需求记录
- 认领记录
- 举报记录
- 申诉记录

## 验收标准

- 从需求、认领到完成形成闭环
- 举报与申诉可被后台完整处理
- 所有处理状态可追踪
