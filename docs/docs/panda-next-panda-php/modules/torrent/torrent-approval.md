# 种子审核

## 功能目标

通过人工或规则化审核控制资源质量，决定资源是否能进入公开列表。

## 参与角色

- 审核管理员
- 发布者

## 入口

- 审核列表页
- 审核详情页
- 审核日志页
- 审核动作接口

## 前台功能

- 发布者查看审核结果
- 根据结果修改并重新提交（如站点允许）

## 后台功能

- 查看待审核资源
- 查看审核详情
- 审核通过
- 审核拒绝
- 填写拒绝原因
- 查看审核日志

## 状态定义

- 待审核
- 审核通过
- 审核拒绝
- 需修改后重提

## 状态流转

1. 资源进入待审核
2. 审核员查看详情
3. 执行通过或拒绝
4. 记录审核日志和拒绝原因
5. 通过后资源公开，拒绝后资源退出公开流程

## 规则与校验

- 审核动作只能由有权限人员执行
- 拒绝必须带原因
- 审核日志不可缺失
- 已完成审核的资源重复审核需有明确策略

## 异常与边界

- 审核对象不存在
- 审核动作重复提交
- 审核人与发布人权限冲突

## 数据结果

- 审核结果
- 拒绝原因
- 审核日志
- 资源公开状态变化

## 验收标准

- 审核通过后资源进入公开列表
- 审核拒绝后前台可见性按规则收敛
- 每次审核动作都有完整日志
