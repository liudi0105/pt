# Tracker Announce

## 功能目标

持续接收客户端上报，维护用户在种子上的实时下载和做种状态。

## 参与角色

- BitTorrent 客户端
- Tracker 服务
- 用户

## 入口

- `/announce`

## 前台功能

- 无页面交互，由客户端自动调用

## 后台功能

- 处理上报结果
- 触发风控和状态更新

## 状态定义

- 新建 Peer
- 下载中
- 做种中
- 已完成
- 已断开
- 客户端异常

## 状态流转

1. 客户端带 passkey 和种子信息发起 announce
2. 系统识别用户、种子、客户端
3. 更新 Peer 在线状态和上传下载量
4. 必要时写入完成事实
5. 返回 tracker 响应

## 规则与校验

- 用户、种子、客户端都必须通过校验
- 非法客户端按风控规则拒绝
- 上报结果必须与用户和种子唯一绑定
- 完成行为必须只在满足条件时生成

## 异常与边界

- passkey 无效
- 种子不存在
- 客户端被拒绝
- 重复完成或断线重连

## 数据结果

- Peer 在线事实
- 上传下载增量
- 完成事实输入
- 风控命中结果

## 验收标准

- announce 可稳定处理新建、更新、完成、断开四类主流程
- 客户端非法时返回明确拒绝结果
- Peer 数据与后续 Snatch、H&R 输入一致
