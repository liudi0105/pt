# 运营资源管理

## 功能目标

集中管理标签、勋章、考核等会被多个业务模块消费的运营资源。

## 参与角色

- 管理员
- 运营人员

## 入口

- 标签管理页
- 勋章管理页
- 考核管理页
- H&R 管理页

## 前台功能

- 用户只消费这些资源的展示和结果

## 后台功能

- 标签 CRUD
- 勋章 CRUD
- 用户勋章 CRUD
- 考核 CRUD
- 考核用户处理
- H&R 处理

## 状态定义

- 资源可用
- 资源停用
- 考核生效/失效
- H&R 待处理/已处理

## 状态流转

1. 运营创建资源
2. 业务模块消费资源
3. 管理员调整生效状态或处理结果

## 规则与校验

- 运营资源修改必须考虑历史数据兼容
- 已被引用的资源删除策略必须明确
- 考核和 H&R 的处理动作必须留痕

## 异常与边界

- 删除已使用标签或勋章
- 停用中的资源仍被前台引用
- 批量处理部分失败

## 数据结果

- 标签
- 勋章
- 考核定义与用户结果
- H&R 处理结果

## 验收标准

- 运营资源具备完整增删改查
- 业务模块能稳定消费这些资源
- 历史记录不会因资源修改而丢失解释能力
