# 客户端风控

## 功能目标

控制允许接入站点的客户端及其例外规则，为下载和 Tracker 行为建立准入门槛。

## 参与角色

- 管理员
- BitTorrent 客户端
- Tracker 服务

## 入口

- Agent Allow 管理页
- Agent Deny 管理页
- Agent Check 接口

## 前台功能

- 无直接前台页面，用户通过客户端使用结果感知风控

## 后台功能

- 白名单规则管理
- 黑名单规则管理
- 客户端校验
- 查看全量规则

## 状态定义

- 允许
- 拒绝
- 例外允许

## 状态流转

1. 管理员维护规则
2. 客户端接入时命中校验
3. 返回允许或拒绝结果
4. Tracker 和下载行为消费该结果

## 规则与校验

- 白名单与黑名单冲突时优先级必须明确
- 例外规则必须可追踪
- 风控结果必须可解释

## 异常与边界

- 客户端标识无法识别
- 规则重复或冲突
- 历史规则误伤正常客户端

## 数据结果

- 规则集合
- 风控命中结果
- 管理审计记录

## 验收标准

- 合法客户端正常通过
- 非法客户端被稳定拒绝
- 规则冲突时结果一致且可解释
