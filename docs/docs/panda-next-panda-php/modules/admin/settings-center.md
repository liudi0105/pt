# 配置中心

## 功能目标

让站点运行参数可被后台管理，而不是散落在代码中或数据库里无约束保存。

## 参与角色

- 管理员
- 系统运行任务

## 入口

- 配置列表页
- 配置详情页

## 前台功能

- 无直接前台页面，前台消费配置结果

## 后台功能

- 查询配置
- 创建配置
- 修改配置
- 删除配置

## 状态定义

- 已生效配置
- 待生效配置（如需要）
- 自动加载配置

## 状态流转

1. 管理员创建或修改配置
2. 系统保存配置值
3. 立即或按策略生效
4. 前台/后台模块读取结果

## 规则与校验

- 配置应按模块分组
- 配置值类型应受控
- 高风险配置变更必须可审计

## 异常与边界

- 配置键重复
- 配置值类型非法
- 删除关键配置导致模块不可用

## 数据结果

- 配置键值
- 自动加载标识
- 配置审计记录

## 验收标准

- 核心站点配置可通过后台完整管理
- 配置生效口径明确
- 配置变更可追溯
