# 配置中心

## 1. 概述

配置中心是站点的统一参数管理模块，让管理员在后台集中控制所有影响前后台行为的运行参数。

配置采用 **key-value 点号命名法** 存储在 `settings` 表（字段：`name`, `value`, `autoload`），支持自动加载缓存（Redis，TTL 600 秒）和实时读取两种生效模式。所有消费方统一通过 `Setting::get('{group}.{key}')` 读取。

覆盖 16 个分组、数百个参数项，分布在两种管理界面中。

## 2. 功能说明

### 2.1 Filament Tab 管理（System > Setting）

以下 5 个分组在后台 `System > Setting` 页面以独立 Tab 管理，每个 Tab 内表单字段按业务语义组织。

#### H&R

H&R（Hit and Run）考核规则控制。

| 配置项 | 说明 |
|-------|------|
| 模式 | 禁用 / 手动（仅指定种子）/ 全局（所有种子启用考核） |
| 考核窗口时长 | 从下载完成开始计时，单位：小时 |
| 最低做种时长 | 达标所需最短做种时间，单位：小时，必须小于考核窗口 |
| 达标分享率 | 分享率达到此值可自动豁免 H&R |
| 封禁阈值 | H&R 数量累计到此值时自动封禁账号 |
| 下载完成率 | 只有下载完成率（0~1）达到此值的种子才计入 H&R |

#### Backup

自动备份配置。

| 配置项 | 说明 |
|-------|------|
| 启用开关 | 是否执行自动备份 |
| 备份频率 | 每天一次 / 每小时一次 |
| 执行时间 | 指定小时和分钟执行 |
| 存储目标 | 支持 Google Drive（client_id / secret / refresh_token / folder_id）、FTP、SFTP |

#### SeedBox

SeedBox（盒子）检测与流量规则。

| 配置项 | 说明 |
|-------|------|
| 启用开关 | 是否启用 SeedBox 规则 |
| 速度阈值 | 非 SeedBox 最大上传速度（Mbps），超过此值且无法匹配 SeedBox 记录时禁用下载 |
| 上传倍率上限 | SeedBox 上传量最多为其种子体积的 N 倍，0 为无限制 |
| 倍率有效期 | 上传倍率在种子发布后 N 小时内有效，0 为始终有效 |
| 免促销开关 | 对 SeedBox 流量不应用促销，上传/下载按实际值计算 |

#### Meilisearch

搜索引擎配置。

| 配置项 | 说明 |
|-------|------|
| 启用开关 | 是否使用 MeiliSearch 作为搜索引擎。启用前需先安装并导入数据 |
| 搜索描述 | 是否同时搜索种子描述。切换后需重新导入数据 |
| 搜索模式 | 精确匹配（不分词）/ AND 分词搜索 |

#### System

系统常规配置。

| 配置项 | 说明 |
|-------|------|
| 改名冷却 | 两次改名之间的最短间隔天数 |
| 改名卡字符限制 | 是否只允许英文字母 |
| 勋章佩戴上限 | 用户最多可同时佩戴的勋章数量 |
| Cookie 有效期 | 登录 Cookie 有效天数 |
| 最大上传速度 | 用于作弊检测（Mbps），实际限速 = 此值 ÷ 检测等级 |
| 邀请预占 | 邀请时是否预占邮箱和用户名，预占后注册时不可更改 |
| 管理后台准入 | 可登录管理后台的最低用户等级 |

### 2.2 传统页面管理（/settings.php）

以下 11 个分组在 `/settings.php` 以表单区块管理。

#### basic

站点基础身份标识，共 3 项：站点名称（SITENAME）、基础 URL（BASEURL）、Tracker announce 地址（announce_url）。

#### main

核心运行参数，共 50+ 项，涵盖以下子类：

- **站点状态**：在线开关、最大用户数、注册策略（开放/关闭/邀请）、邀请系统、验证方式、未确认注册过期时间
- **Tracker 间隔**：announce 间隔（默认 30 分钟）、第二/第三区间生效条件和间隔
- **种子与上传**：种子文件大小上限（1MB）、存储目录、每页种子数、初始上传量、种子文件名前缀
- **Offer 系统**：模块开关、通过投票数、投票超时、上传超时、自动通过阈值
- **界面展示**：帖子/主题/种子每页数量、新闻条数、首页模块开关（热门/经典/IMDb/NFO/统计/Shoutbox/Funbox/帮助）、样式/语言
- **自动清理**：5 级清理间隔（60 分钟 ~ 15 天）
- **其他**：死种超时（6 小时）、站点邮箱/捐赠账号/备案号、PT-Gen、全局搜索、周末免费、多语言等

#### smtp

邮件发送配置，共 10 项：SMTP 类型（默认/高级/外部）、通知开关、主机端口、加密方式、账号密码。

#### security

安全策略，共 17 项：SSL（Web 和 Tracker）、验证码、同一 IP 最大注册数、登录尝试限制、作弊检测等级（1~4）、免检测用户等级、访客访问模式（正常/静态页/自定义/重定向）、登录密钥。

#### authority

权限映射，共 40+ 项，每种操作对应一个最低用户等级：

| 操作类型 | 示例 |
|---------|------|
| 前台操作 | 上传种子（User）、匿名发布（PowerUser）、请求补种（User）、购买邀请（PowerUser） |
| 内容管理 | 新闻管理（Sysop）、帖子管理（Admin）、评论管理（Admin）、论坛管理（Sysop） |
| 种子管理 | 种子管理（Admin）、种子审核（Sysop）、种子删除（Sysop）、设置 H&R（Sysop） |
| 用户管理 | 用户删除（Sysop）、变更等级（Sysop）、查看完整资料（Sysop） |

#### tweak

杂项配置，共 14 项：积分系统开关、地理位置、IP 记录、SEO 关键词/描述、SQL 调试、Tooltip、分析代码。

#### bonus

积分（Karma Bonus Point）经济体系，共 35+ 项：

- **做种积分公式**：基础积分/种子、种子数上限、公式参数（T0/N0/B0/L）、官方/后宫/零积分加成
- **行为奖励**：发布种子/字幕、论坛发帖/回帖、评论、投票、Offer 投票、感谢等分别奖励积分
- **兑换比例（上传量）**：1GB/5GB/10GB/100GB 上传量分别所需积分
- **兑换比例（下载量）**：10GB/100GB 下载量分别所需积分
- **兑换比例（其他）**：邀请、临时邀请、VIP、自定义头衔、改名卡、彩虹 ID、取消 H&R、签到卡
- **兑换限制**：最低分享率（6）和上传量（50GB）
- **积分转账**：转账开关、固定税、百分比税、推广链接奖励

#### account

账号升降级与禁用规则，共 30+ 项：

- **禁用规则**：永不删除的最低等级、封存/未封存账号不活跃天数阈值、Peasant 禁用天数、物理删除开关
- **等级晋升条件**：从 PowerUser 到 Nexus Master 共 7 个等级，每个等级需要达到的注册时长（周）、上传量（GB）、分享率
- **Peasant 降级条件**：5 级下载量 + 分享率阈值，逐级降低
- **晋升奖励邀请数**：各等级晋升时奖励的邀请数量

#### torrent

种子促销与种子认领，共 35+ 项：

- **随机促销概率**：上传种子时触发 50% 下载/免费/2x 上传/2x 免费等促销的百分比概率
- **大体积种子促销**：大于 N GB 的种子自动应用指定促销
- **促销过期时间**：各类促销（50% 下载/免费/2x 上传/2x 免费/30% 下载）的过期天数
- **类型转换**：每种促销到期后转换为何种类型
- **自动热种**：窗口天数和做种人数阈值
- **置顶样式**：一/二级置顶背景色
- **种子认领（Claim）**：认领开关、认领有效期/人数上限/积分扣除/做种加成/达标标准
- **付费种子**：开关、税率、最高价格
- **下载与审核**：passkey 要求、审核状态图标、NFO 视图样式

#### attachment

附件系统，共 30+ 项：

- **四级权限**：按用户等级分 4 档，每档配置上传数量上限（每 24h）、单文件大小上限（KB）、允许的扩展名
- **存储路径**：目录位置、HTTP 路径、目录组织方式（按月/按天/平铺）
- **缩略图**：生成模式、质量、尺寸（常规和备用）
- **水印**：位置（9 宫格）、最小图片尺寸、质量

#### advertisement

广告系统，共 8 项：广告开关、用户去广告（等级条件 + 积分购买，含价格和有效期）、广告点击奖励。

#### code

版本信息，共 4 项，只读：主版本名、子版本号、发布日期、官方网站。

## 3. 操作入口

| 入口 | 路径 | 管理范围 |
|------|------|---------|
| Filament 后台 | System > Setting | hr, backup, seed_box, meilisearch, system |
| 传统设置页 | /settings.php | basic, main, smtp, security, authority, tweak, bonus, account, torrent, attachment, advertisement, code |
| API | `resource /api/settings` | 通用增删改查，支持分组前缀过滤和批量写入 |

## 4. 生效机制

配置的生效时间取决于 `autoload` 标记：

| 类型 | 条件 | 读取方式 | 修改后生效 |
|------|------|---------|-----------|
| 自动加载 | `autoload=yes` | 缓存在 Redis（键 `nexus_settings_in_laravel`，TTL 600 秒），全量加载 | 修改后清除缓存，下次读取新值 |
| 非自动加载 | `autoload=no` | 每次消费时直接 `SELECT` 单条查询 | 下次读取立即生效 |

修改流程：提交表单 → `Setting::upsert()` 批量写入 → 清除 Redis 和遗留缓存 → `nexus_setting_update` action 通知扩展 → 下次读取新值。
