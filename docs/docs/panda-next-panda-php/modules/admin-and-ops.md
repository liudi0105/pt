# 后台与治理

## 模块目标

后台与治理模块负责站点的配置、风控、统计、字典和插件生命周期管理。

它是整个站点的控制面板。

## 功能清单

- 站点配置
- 系统字典维护
- 用户与种子管理
- 勋章、考核、H&R、SeedBox 管理
- 客户端 allow / deny 控制
- 上传和下载速度规则
- ISP 与网络环境管理
- 插件管理
- 仪表盘统计

## 子功能详情

### 1. 站点配置中心

站点很多行为都应该从配置中心读取，而不是硬编码。

需要支持：

- 全局开关
- 模块开关
- 运营参数
- 安全参数
- 搜索参数
- 备份参数

### 2. 字典与基础资料

站点需要一组稳定的基础枚举和字典。

包括：

- 分类
- 编码
- 音轨
- 来源
- 处理方式
- 站点分区
- 状态枚举

### 3. 风控和访问治理

风控是 PT 站生存的底线。

需要支持：

- 客户端白名单和黑名单
- 上传/下载速度规则
- SeedBox 识别
- ISP 规则
- 异常行为处理

### 4. 插件生命周期

插件不是附属功能，而是站点能力扩展方式。

需要支持：

- 插件安装
- 插件启用和禁用
- 插件升级
- 插件卸载
- 插件权限和菜单扩展

### 5. 运营仪表盘

后台需要给运营人员一个快速感知站点状态的界面。

常见指标包括：

- 用户增长
- 种子增长
- 做种活跃度
- 考核完成情况
- H&R 处理情况
- 最新活动

## 关键规则

- 配置变更必须立即影响站点运行态
- 风控规则必须可解释、可追溯、可回滚
- 插件必须具备完整生命周期管理
- 所有人工操作必须留痕

## 关键输出

- 站点配置结果
- 风控命中结果
- 插件状态
- 后台统计面板
- 基础字典

## 依赖模块

- 账户与权限
- 数据与存储
- 种子与 Tracker
- 社区与运营

## 验收标准

- 管理员可以完成核心配置和字典维护
- 风控规则能够生效并影响站点行为
- 插件能完整完成生命周期操作
- 仪表盘能够反映站点关键状态
