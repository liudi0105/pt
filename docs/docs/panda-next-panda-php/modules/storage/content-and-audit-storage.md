# 内容互动与审计存储

## 1. 概述

承载用户前台互动记录（评论、感谢、书签、消息、通知、签到）和后台治理审计日志（操作记录、禁用日志、更名日志、积分日志、登录日志），保证业务可追溯和可解释。

## 2. 功能说明

### 2.1 互动记录表

| 表 | 用途 | 说明 |
|----|------|------|
| `comments` | 种子评论 | 关联 `users`(评论者) + `torrents` |
| `thanks` | 感谢记录 | 用户感谢种子的记录 |
| `bookmarks` | 书签收藏 | 用户收藏种子 |
| `messages` | 站内信 | 用户间通信和系统通知 |
| `notifications` | 系统通知 | 站内提醒 |
| `attendance_logs` | 签到记录 | 每日签到历史 |

互动记录支持前台展示，删除操作通常为软删除或隐藏，不直接物理删除。

### 2.2 审计日志表

| 表 | 用途 | 关键字段 |
|----|------|----------|
| `sitelog` | 系统审计日志 | `security_level` 控制可见范围 |
| `torrent_operation_logs` | 种子管理操作 | 操作人、种子 ID、操作类型、变更内容 |
| `user_ban_logs` | 用户禁用事件 | 操作人、目标用户、原因、时间 |
| `username_change_logs` | 用户名变更历史 | 原用户名、新用户名、操作人 |
| `bonus_logs` | 积分交易日志 | 20+ 业务类型标识，记录收入/支出 |
| `login_logs` | 登录历史 | 用户 ID、IP、地理位置、User-Agent |

### 2.3 bonus_logs 业务类型
积分日志通过 `business_type` 字段区分不同来源，目前支持 20+ 类：
- `bonus_harvest` — 做种收获
- `bonus_exchange` — 积分兑换
- `bonus_reward` — 奖励发放
- `bonus_penalty` — 违规扣减
- `bonus_transfer` — 积分转账

### 2.4 分层原则
- **互动层**：用户可见，支持前台操作（发表、隐藏、删除）。
- **审计层**：仅管理员可见，不可删除或修改。
- 删除前台内容（如评论隐藏）不会删除审计痕迹。
- 审计记录包含：操作人、操作对象、操作时间、变更前后内容、操作结果。

### 2.5 数据关系
```
用户操作 → 互动表（前台展示）
         → 审计表（后台追溯）
```
两者独立写入，独立查询，互不干扰。

## 3. 操作入口

- 互动操作：前台页面（评论发布、书签添加、签到等）
- 审计查询：Filament 后台各管理页面
- 积分日志：用户中心 + Filament 后台

## 4. 使用说明

1. 用户删除评论应仅标记为隐藏/删除状态，审计日志中保留原始内容。
2. 积分业务类型需在 `bonus_logs.business_type` 中使用已注册类型，便于统计。
3. 登录日志用于安全分析和异常检测，不应被删除或截断。
4. 用户名变更日志保留所有历史记录，便于用户身份追溯。
5. 审计日志的 `security_level` 控制谁可以查看，敏感操作需较高等级。

## 5. 配置参考（可选）

- `sitelog` 的 `security_level` 阈值需与用户等级体系对齐
- 积分业务类型枚举集中定义在 `config/bonus.php` 或对应 Service 中
