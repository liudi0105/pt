# Tracker 运行态存储

## 功能目标

承载高频的下载、做种和连接状态更新，避免污染主业务查询模型。

## 覆盖对象

- Peer 在线状态
- Announce 更新结果
- Scrape 统计输入
- Snatch 生成前后的运行态事实

## 存储原则

- 高频写入与主业务查询分开考虑
- 在线态与历史态分层
- 运行态允许过期，但历史事实不能丢失

## 关键规则

- Peer 是运行态
- Snatch 是历史态
- Tracker 高频写入不能阻塞主页面访问

## 验收标准

- Tracker 数据层可支撑持续高频更新
- Peer 和 Snatch 的职责边界清晰
