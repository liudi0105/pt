# 附件、文件与备份

## 1. 概述

管理站点文件资源的存储、引用和备份，包括 .torrent 种子文件、附件（截图/图片）以及数据库和文件的灾备方案。

## 2. 功能说明

### 2.1 .torrent 文件存储
种子文件以 `.torrent` 格式存储于磁盘，文件路径为 `{$torrent_dir}/{$id}.torrent`：
- `torrent_dir` 通过站点设置配置
- 文件名关联 `torrents` 表的 `id` 字段
- 上传时生成、删除种子时移除（业务中可配置是否保留文件）

### 2.2 附件存储
附件系统支持可配置存储位置（`savedirectory` 设置）：
- 按月份组织目录结构：`{savedirectory}/{YYYYMM}/`
- 支持缩略图生成（大图自动压缩）
- 支持水印添加（配置开启）
- 元数据入库，文件实体存储在磁盘
- 引用关系：附件关联到具体业务对象（种子截图、用户头像等）

### 2.3 数据库备份
通过系统备份配置定期执行：
- 完整数据库 dump
- 支持定时自动备份
- 备份文件包含完整表结构和数据

### 2.4 文件备份
- 涵盖数据库 dump + 业务文件（种子文件、附件、配置文件）
- 支持多种备份目标：
  - Google Drive
  - FTP 服务器
  - SFTP 服务器

### 2.5 恢复机制
- 数据库恢复：使用数据库 dump 重建数据
- 文件恢复：从备份目标拉取文件到对应目录
- 恢复验证：定期检查备份文件的完整性和可恢复性

## 3. 操作入口

- 种子上传：发布种子页面（传统 PHP）
- 附件管理：Filament 后台附件管理
- 备份配置：Filament 后台 → 备份设置
- 备份运行：系统调度任务 / 手动执行
- 恢复操作：通过命令行或运维工具执行

## 4. 使用说明

1. 种子文件路径固定，删除种子时注意确认文件是否保留。
2. 附件按月分目录存储，避免单目录文件过多。
3. 备份必须同时覆盖数据库和业务文件两个层面，单一备份不可靠。
4. 备份目标建议配置至少两个（本地+远程），防止单点故障。
5. 文件删除操作需校验业务引用，防止误删正在使用的文件。

## 5. 配置参考（可选）

- `torrent_dir` — 种子文件存储根目录
- `savedirectory` — 附件存储根目录
- 备份频率：由调度任务 CRON 表达式控制
- 备份目标：系统中配置的 Google Drive / FTP / SFTP 凭据
